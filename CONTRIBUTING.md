# Contributing to ShopKart

Thank you for your interest in contributing! This guide covers how to get set up and submit changes.

---

## Getting Started

1. **Fork** the repository and clone your fork locally.
2. Follow the [setup instructions in the README](README.md#-getting-started) to run the project.
3. Create a new branch for your change:
   ```bash
   git checkout -b feat/your-feature-name
   ```

---

## Types of Contributions

- **Bug fixes** — open an issue first if the bug isn't already reported
- **New features** — open an issue to discuss before implementing
- **Documentation** — typo fixes, clearer explanations, missing steps
- **Tests** — additional coverage is always welcome

---

## Pull Request Guidelines

- Keep PRs focused — one feature or fix per PR
- Write a clear PR description explaining **what** changed and **why**
- Make sure the app runs without errors before submitting
- Follow the existing code style (ESLint/Prettier if configured)

---

## Reporting Issues

When opening a bug report, include:
- Steps to reproduce
- Expected vs actual behavior
- Your Node.js and npm versions
- Any relevant error messages or screenshots

---

## Code of Conduct

Be respectful. Constructive feedback is welcome; harassment is not.
